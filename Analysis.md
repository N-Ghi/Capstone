# Capstone Project Analysis

## Aim of the project

The project was concieved to be a platform that acts as a mediator between local tourism operators in Rwanda and tourism; both local and international.

Part of the problems addressed include language barrier and digital visibility.

To achieve the aim of the project, the platform needed to have select features that directly addressed the problems.

## How the project achieves its aim

1. **Language Barrier**

    To address the problem of language barrier, Azure's text translate API was used to change user generated text between supported languages. Hard-coded i18n json files are used for system text since they are predetermined and can be assigned translations manually.

2. **Digital Visibility**

    The project addresses digital visibility by creating a centralized digital marketplace where guides can publish and promote their services online.
    This approach attempts to re-imagine the discovery of tourism experiences, stepping away from the traditional word-of-mouth, physical tour offices and the more recent social media promotions.
    Through the platform, tourists can discover, book, and pay for experiences online, giving them the ability to plan their itineraries as they wish. Guides can also become visible to a global audience who would otherwise be unable to discover them.

3. **Location Finding**

    The use of Google's Map API was used to allow guides to select real-life locations that can be rendered to produce a map. Adding a "Get Directions" options would allow tourists who are not familiar with the area to get there with no issue.

4. **Establish measurable metrics that recognise the impact and limitations of the platform.**

    Several evaluation metrics were identified:

    |Metric                         | Observation                                            |
    | ------------------------------|--------------------------------------------------------|
    | **Experience discoverability**| All uploaded experiences appeared in discovery listings|
    | **Language accessibility**    |Content can be viewed in multiple languages             |
    | **Platform inclusivity**      | Operators can independently publish experiences        |
